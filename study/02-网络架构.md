# 02 - 网络架构详解

> 📚 学习目标：理解 TransformNet 和 VGG 的结构设计原理

## 一、整体架构

Fast Neural Style 包含两个网络：

```
训练阶段：
┌─────────────────────────────────────────────────────────┐
│                                                         │
│  内容图像 → TransformNet → 生成图像                      │
│                              ↓                          │
│                         VGG (冻结)                       │
│                              ↓                          │
│                    提取特征 (多层)                        │
│                              ↓                          │
│              计算 content_loss + style_loss              │
│                              ↓                          │
│                    反向传播更新 TransformNet              │
│                                                         │
└─────────────────────────────────────────────────────────┘

推理阶段：
新图像 → TransformNet → 风格化图像 (VGG 不再需要)
```

## 二、TransformNet（图像转换网络）

### 2.1 设计目标

输入和输出都是图像，分辨率相同，但风格不同。

**设计要求**：
1. 保持图像分辨率（不能下采样后不恢复）
2. 提取足够的语义信息（需要较深的网络）
3. 生成高质量图像（需要上采样层）
4. 保留细节（使用跳跃连接或残差结构）

### 2.2 网络结构（编码器-残差-解码器）

```
输入图像 (3, 256, 256)
    ↓
┌─────────────────────────┐
│   下采样层 (Encoder)      │
│   - Conv 3→32, stride=1  │
│   - Conv 32→64, stride=2 │
│   - Conv 64→128, stride=2│
└─────────────────────────┘
    ↓ (128, 64, 64)
┌─────────────────────────┐
│   残差层 (Residual)      │
│   - ResBlock × 5         │
│   - 保持尺寸不变          │
└─────────────────────────┘
    ↓ (128, 64, 64)
┌─────────────────────────┐
│   上采样层 (Decoder)     │
│   - Upsample + Conv 128→64│
│   - Upsample + Conv 64→32 │
│   - Conv 32→3, stride=1   │
└─────────────────────────┘
    ↓
输出图像 (3, 256, 256)
```

### 2.3 各部分详解

#### 2.3.1 下采样层（Downsampling Layers）

**作用**：提取抽象特征，扩大感受野

```python
# 第一层：保持尺寸，增加通道
Conv2d(3, 32, kernel_size=9, stride=1, padding=4)
InstanceNorm2d(32)
ReLU(inplace=True)

# 第二层：尺寸减半
Conv2d(32, 64, kernel_size=3, stride=2, padding=1)
InstanceNorm2d(64)
ReLU(inplace=True)

# 第三层：尺寸再减半
Conv2d(64, 128, kernel_size=3, stride=2, padding=1)
InstanceNorm2d(128)
ReLU(inplace=True)
```

**关键设计**：
- 第一层使用大卷积核（9×9）：捕捉更大的纹理模式
- 后续使用小卷积核（3×3）：标准设计，平衡效果和计算量
- stride=2：每次将尺寸减半，通道数翻倍
- InstanceNorm：关键技巧（下文详解）

#### 2.3.2 残差层（Residual Blocks）

**作用**：深层特征变换，同时保持梯度流动

残差块的结构：
```
输入 x
  ↓
Conv 128→128, 3×3, padding=1
  ↓
InstanceNorm + ReLU
  ↓
Conv 128→128, 3×3, padding=1
  ↓
InstanceNorm
  ↓
输出 = x + 上面的结果  ← 跳跃连接
```

**为什么使用残差块？**

1. **梯度流动更好**：
   - 普通网络：梯度需要经过所有层
   - 残差网络：梯度可以直接通过跳跃连接传递
   
2. **学习恒等映射**：
   - 如果某些特征不需要变换，网络可以学到"什么都不做"（输出=输入）
   - 这在风格迁移中很有用：有些内容特征应该保持不变

3. **允许更深的网络**：
   - 可以堆叠5个甚至更多残差块
   - 不用担心梯度消失

#### 2.3.3 上采样层（Upsampling Layers）

**作用**：恢复图像分辨率

有两种实现方式：

**方式1：转置卷积（Transposed Convolution）**
```python
ConvTranspose2d(128, 64, kernel_size=3, stride=2, padding=1, output_padding=1)
```
- 优点：可学习的上采样
- 缺点：容易产生"棋盘效应"（checkerboard artifacts）

**方式2：上采样 + 卷积（推荐）**
```python
Upsample(scale_factor=2, mode='nearest')  # 或 'bilinear'
Conv2d(128, 64, kernel_size=3, stride=1, padding=1)
```
- 优点：更稳定，伪影更少
- 缺点：计算量稍大

**完整上采样流程**：
```python
# 第一次上采样：64×64 → 128×128
Upsample(scale_factor=2)
Conv2d(128, 64, kernel_size=3, stride=1, padding=1)
InstanceNorm2d(64)
ReLU(inplace=True)

# 第二次上采样：128×128 → 256×256
Upsample(scale_factor=2)
Conv2d(64, 32, kernel_size=3, stride=1, padding=1)
InstanceNorm2d(32)
ReLU(inplace=True)

# 输出层：保持尺寸，通道变为3
Conv2d(32, 3, kernel_size=9, stride=1, padding=4)
Tanh()  # 输出范围 [-1, 1]
```

### 2.4 Instance Normalization（实例归一化）

**这是 Fast Neural Style 的关键技巧！**

#### 2.4.1 什么是 Instance Normalization？

对每个样本的每个通道独立归一化：

$$
y_{n,c,h,w} = \frac{x_{n,c,h,w} - \mu_{n,c}}{\sqrt{\sigma_{n,c}^2 + \epsilon}}
$$

- $n$：batch 中的第几个样本
- $c$：第几个通道
- $\mu_{n,c}$：该样本该通道的均值
- $\sigma_{n,c}$：该样本该通道的标准差

#### 2.4.2 与 Batch Normalization 的区别

| 特性 | Batch Normalization | Instance Normalization |
|------|-------------------|----------------------|
| 归一化维度 | 在 batch 维度上统计 | 在每个样本内部统计 |
| 依赖关系 | 依赖 batch 内其他样本 | 只依赖当前样本 |
| 适用场景 | 分类任务 | 风格迁移、图像生成 |

#### 2.4.3 为什么对风格迁移有效？

**直觉解释**：
- 图像的对比度信息（均值和方差）本身就是"风格"的一部分
- Instance Norm 去除了原始图像的对比度
- 让网络专注于学习**新的**对比度和纹理模式

**实验证明**：
- 使用 Instance Norm：效果好，收敛快
- 使用 Batch Norm：效果较差
- 不使用归一化：效果最差

## 三、VGG 网络（特征提取器）

### 3.1 VGG 结构

VGG16 的结构（我们使用预训练权重）：

```
输入图像 (3, H, W)
    ↓
Block 1:
  conv1_1: Conv2d(3, 64, 3×3)
  relu1_1: ReLU
  conv1_2: Conv2d(64, 64, 3×3)
  relu1_2: ReLU  ← 用于 style loss
  pool1: MaxPool2d(2×2)
    ↓
Block 2:
  conv2_1: Conv2d(64, 128, 3×3)
  relu2_1: ReLU
  conv2_2: Conv2d(128, 128, 3×3)
  relu2_2: ReLU  ← 用于 style loss
  pool2: MaxPool2d(2×2)
    ↓
Block 3:
  conv3_1: Conv2d(128, 256, 3×3)
  relu3_1: ReLU
  conv3_2: Conv2d(256, 256, 3×3)
  relu3_2: ReLU
  conv3_3: Conv2d(256, 256, 3×3)
  relu3_3: ReLU  ← 用于 style loss
  pool3: MaxPool2d(2×2)
    ↓
Block 4:
  conv4_1: Conv2d(256, 512, 3×3)
  relu4_1: ReLU
  conv4_2: Conv2d(512, 512, 3×3)
  relu4_2: ReLU  ← 用于 content loss
  conv4_3: Conv2d(512, 512, 3×3)
  relu4_3: ReLU  ← 用于 style loss
  pool4: MaxPool2d(2×2)
    ↓
Block 5: (可选，不一定用)
  conv5_1, relu5_1, conv5_2, relu5_2, conv5_3, relu5_3
```

### 3.2 选择哪些层？

#### 3.2.1 Content Loss 的层选择

**通常选择：relu4_2**

原因：
- 深层特征：包含高层语义信息
- 不太深：不会完全丢失细节
- 实验验证效果好

也可以尝试：
- relu3_3：保留更多细节，但可能不够抽象
- relu5_2：更抽象，但可能丢失太多细节

#### 3.2.2 Style Loss 的层选择

**通常选择：relu1_2, relu2_2, relu3_3, relu4_3**

原因：
- 多个层级：捕捉不同尺度的风格特征
- relu1_2：细粒度纹理（笔触、小波纹）
- relu2_2：中等纹理（色块）
- relu3_3：大尺度模式
- relu4_3：全局色调

### 3.3 VGG 的预处理

VGG 在 ImageNet 上训练时使用的归一化：

```python
mean = [0.485, 0.456, 0.406]  # RGB 通道的均值
std = [0.229, 0.224, 0.225]   # RGB 通道的标准差

# 对输入图像归一化
normalized = (image - mean) / std
```

**重要**：
- TransformNet 的输出范围是 [-1, 1]（因为使用 Tanh）
- 需要转换到 [0, 1]，然后再用 VGG 归一化
- 或者，让 TransformNet 直接输出 [0, 1] 范围（不用 Tanh）

## 四、参数量分析

### 4.1 TransformNet 参数量

粗略估计：
- 下采样层：约 0.1M 参数
- 残差块（5个）：约 1.5M 参数
- 上采样层：约 0.1M 参数
- **总计**：约 **1.7M 参数**

相比 VGG16（138M 参数），TransformNet 非常轻量！

### 4.2 内存占用

训练时（batch_size=4, image_size=256）：
- 输入：4 × 3 × 256 × 256 = 0.75 MB
- 中间特征图：约 50-100 MB
- 梯度：与参数量相同，约 7 MB
- VGG 特征（不需要梯度）：约 20-30 MB
- **总计**：约 **100-150 MB**（非常小！）

RTX 4090 D (24GB) 可以轻松处理 batch_size=8 甚至 16。

## 五、设计哲学

### 5.1 为什么是这个结构？

**编码器-解码器**：
- 图像到图像的标准架构
- 编码器提取特征，解码器重建图像

**残差连接**：
- 保持内容信息（有些特征不应该改变）
- 使深层网络可训练

**Instance Norm**：
- 专门为风格迁移设计
- 实验证明效果最好

### 5.2 可以改进的地方

**可以尝试**：
- 更多残差块（5 → 7 或 9）
- 更大的网络（通道数翻倍）
- 注意力机制（在某些区域应用更强的风格）
- 深度可分离卷积（减少参数量）

**但要注意**：
- 太大的网络：训练慢，推理慢
- 太复杂的结构：可能过拟合
- 实验时间有限：先用标准结构

## 六、总结

### 核心设计原则

1. **TransformNet**：轻量、快速、端到端
   - 编码器-残差-解码器结构
   - Instance Normalization 是关键
   - 输入输出尺寸相同

2. **VGG**：强大、预训练、冻结参数
   - 提供层次化的特征表示
   - 多层输出用于计算损失
   - 推理时不需要

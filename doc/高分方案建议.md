# 图像风格迁移实验高分方案建议

更新时间：2026-06-18

## 一、总体判断

如果目标只是完成实验，使用预训练 Fast Neural Style 模型进行 CPU 推理即可较快产出结果。但如果目标是期末拿高分，仅使用预训练模型的分数上限不高。

根据 `实验要求.md`，本实验多次强调：

- 使用 ImageNet 数据集训练图像风格迁移模型。
- 需要训练出一个能够实现图像风格迁移的模型。
- 训练模型并记录损失值。
- 使用训练好的模型测试并记录 `delta_time`。
- 报告中必须包含模型详情、模型亮点、损失值、迁移耗时和结果分析。

因此，高分方案应优先选择：

```text
PyTorch 自己复现 Fast Neural Style
+ 使用 ImageNet 子集或小型内容图数据集训练
+ 自选 1 张风格图
+ 训练一个单风格前馈模型
+ 记录 content_loss / style_loss / total_loss
+ 测试多张自己的内容图
+ 记录 delta_time
+ 做参数或方法对照实验
```

一句话建议：

> 用 PyTorch 自己复现 Fast Neural Style，并用云服务器 GPU 训练一个单风格模型；本地 CPU 只作为保底推理和报告验证。

## 二、为什么不建议只做 CPU 预训练模型推理

只用 CPU 跑预训练模型的优点是快、稳定、容易出图，但它更像“复现和评估已有模型”，不太满足实验要求中的“训练模型”部分。

这种方案适合作为保底方案，但不适合作为高分主路线。

主要风险：

1. 训练过程不足，无法展示自己训练出的模型。
2. 损失曲线、训练日志、中间结果较少。
3. 报告容易变成对现成项目的运行说明。
4. 如果老师严格按照“需要训练出模型”评分，分数上限会受影响。

## 三、为什么建议使用云服务器训练

Fast Neural Style 从头训练需要较多图像数据和较长训练时间。CPU 虽然可以跑，但训练速度慢，调参成本高，容易导致训练轮数太少、结果质量不稳定。

云服务器或 GPU 环境的价值在于可以稳定获得高分证据：

- 真实训练过程。
- `content_loss`、`style_loss`、`total_loss` 的下降曲线。
- 自己训练得到的 `.pth` 模型文件。
- 多轮训练日志。
- 训练中间样例图。
- 推理速度测试。
- 参数调整与对照分析。

这些材料比单纯展示几张预训练输出图更能体现实验工作量和独立性。

## 四、推荐方案

### 方案名称

PyTorch Fast Neural Style 单风格模型训练与性能评估。

### 实验定位

本实验基于 PyTorch 复现 Fast Neural Style 的前馈图像风格迁移方法，使用自选风格图和内容图训练集训练一个单风格图像转换网络，并通过损失曲线、结果图、迁移耗时和对照实验评估模型效果。

### 技术路线

```text
内容图像数据集
    ↓
图像转换网络 TransformNet
    ↓
生成风格化图像
    ↓
VGG16/VGG19 特征提取网络
    ↓
计算 content_loss / style_loss / tv_loss
    ↓
反向传播更新 TransformNet
    ↓
保存训练好的模型
    ↓
测试新内容图像并记录 delta_time
```

### 推荐模型结构

图像转换网络：

- 卷积层。
- 下采样层。
- 5 个残差块。
- 上采样层。
- 输出卷积层。
- Instance Normalization。
- ReLU / Tanh 激活。

损失网络：

- 使用 ImageNet 预训练 VGG16 或 VGG19。
- 冻结 VGG 参数。
- 使用中间层特征计算内容损失和风格损失。

损失函数：

- 内容损失 `content_loss`。
- 风格损失 `style_loss`。
- 总变差损失 `tv_loss`。
- 总损失 `total_loss`。

## 六、分阶段执行计划

### 阶段 1：本地搭建 PyTorch 项目骨架

目标：先把代码结构和最小训练流程搭起来。

需要完成：

1. 创建项目目录。
2. 创建 `requirements.txt`。
3. 创建 `config.py`。
4. 实现 `models/transform_net.py`。
5. 实现 `models/vgg.py`。
6. 实现 `utils/dataset.py`。
7. 实现 `utils/loss.py`。
8. 编写 `train.py` 和 `test.py`。

阶段验收标准：

- 模型能够完成前向传播。
- VGG 能提取特征。
- 损失函数能够计算。
- 小 batch 能完成一次反向传播。

### 阶段 2：本地小数据跑通训练

目标：证明完整训练流程可用。

建议数据：

- 内容图：几十到几百张即可。
- 风格图：1 张。
- 图像尺寸：`128` 或 `256`。
- batch size：`2` 或 `4`。
- epoch：`1`。

阶段验收标准：

- 能输出 `content_loss`、`style_loss`、`total_loss`。
- 能保存 checkpoint。
- 能生成一张测试结果图。
- 能记录一次 `delta_time`。

### 阶段 3：云服务器 GPU 正式训练

目标：获得高质量训练结果和正式实验数据。

推荐配置：

```text
GPU：T4 / RTX 3060 / RTX 3090 / A10 均可
显存：8GB 以上
系统：Ubuntu
框架：PyTorch
训练数据：1000-5000 张内容图
图像尺寸：256
batch size：4-8
epoch：2-4
```

推荐训练参数：

```text
learning_rate = 1e-3
content_weight = 1.0
style_weight = 1e5
tv_weight = 1e-6
image_size = 256
batch_size = 4 或 8
epochs = 2-4
```

需要保存：

- `checkpoints/best_model.pth`
- `logs/train_loss.csv`
- `results/loss_curves.png`
- `results/samples/epoch_*.png`
- 训练命令和环境记录。

阶段验收标准：

- 损失整体有下降趋势。
- 输出图像具有目标风格。
- 内容结构仍可辨认。
- 训练过程可复现。

### 阶段 4：测试与性能评估

目标：使用训练好的模型测试多张自己的内容图，并记录指标。

建议测试图像：

- 校园或建筑图。
- 自然风景图。
- 人物、物品或室内场景图。

每张图记录：

- 输入尺寸。
- 输出尺寸。
- `content_loss`。
- `style_loss`。
- `total_loss`。
- `delta_time`。
- 推理设备。

推荐表格：

| 测试图 | content_loss | style_loss | total_loss | delta_time/s | 效果评价 |
|---|---:|---:|---:|---:|---|
| campus.jpg | 实测 | 实测 | 实测 | 实测 | 建筑结构保留较好 |
| landscape.jpg | 实测 | 实测 | 实测 | 实测 | 天空和树木纹理明显 |
| object.jpg | 实测 | 实测 | 实测 | 实测 | 主体轮廓清晰 |

### 阶段 5：增加对照实验

高分报告不能只展示“跑通了”，最好做一个控制变量实验。

最推荐对照：不同 `style_weight` 对比。

例如：

```text
style_weight = 1e4
style_weight = 1e5
style_weight = 1e6
```

分析角度：

- 风格权重较小时，内容保持更好，但风格不明显。
- 风格权重较大时，纹理更强，但细节更容易丢失。
- 中间权重通常能取得更好的平衡。

可选对照：

- 不同分辨率对比：`128`、`256`、`512`。
- CPU 与 GPU 推理时间对比。
- 不同测试内容类型对比。

## 八、方案对比

| 方案 | 分数上限 | 风险 | 是否推荐 |
|---|---:|---:|---|
| 只借鉴同学报告改写 | 很低 | 很高 | 不推荐 |
| 本地 CPU 跑预训练模型 | 中等 | 低 | 保底方案 |
| PyTorch 自己复现 + CPU 小训练 | 中上 | 中 | 可行 |
| PyTorch 自己复现 + 云服务器 GPU 训练 | 最高 | 中 | 最推荐 |
| Torch7 原项目复现 | 中等 | 高 | 不推荐作为主线 |

## 九、保底方案

如果云服务器训练遇到问题，至少完成：

1. 本地或云端跑通预训练模型推理。
2. 使用自己的内容图像。
3. 生成 2-3 种风格迁移结果。
4. 记录 `delta_time`。
5. 使用 VGG 评估脚本计算 `content_loss` 或 `style_loss`。
6. 报告中如实说明使用预训练模型进行推理评估。

这个方案能保证完成实验，但高分概率低于自己训练模型。

## 十、下一步最应该做什么

立刻开始以下 5 件事：

1. 确定使用 PyTorch 自己复现，不再以 Torch7 原项目作为主线。
2. 准备 1 张目标风格图。
3. 准备 1000 张左右训练内容图，数据不足时先用 100-300 张跑通。
4. 本地先完成一次小 batch 训练，验证代码正确。
5. 再上云服务器进行正式训练，保存模型、日志、曲线和结果图。

当前第一目标不是写报告，而是跑通最小训练闭环：

```text
数据加载 → TransformNet 前向传播 → VGG 特征提取 → 损失计算 → 反向传播 → 保存模型 → 测试输出图
```

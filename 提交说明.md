# 图像风格迁移项目提交说明

本说明用于课程提交前整理项目文件。原则是：核心代码、报告、关键结果必须交；原始数据集、过程归档和工具配置不交；如果老师要求“项目可运行”，再补最小测试数据和数据准备说明。

## 一、建议提交的内容

| 内容 | 建议 | 说明 |
|---|---|---|
| `models/` | 必交 | TransformNet 与 VGG16 特征提取器，核心模型实现 |
| `utils/` | 必交 | 数据集、图像预处理、Gram 矩阵、感知损失等工具函数 |
| `train.py` | 必交 | 训练入口，含 best-checkpoint、验证图评估、CSV 日志 |
| `test.py` | 必交 | 推理与测试入口，记录 `delta_time`、loss 等指标 |
| `test_stretch.py` | 建议提交 | 报告中讨论尺寸/拉伸实验，保留该脚本更完整 |
| `config.py` | 必交 | 默认路径和基础参数配置 |
| `requirements.txt` | 必交 | 环境依赖 |
| `README.md` | 必交 | 项目说明、复现步骤、数据准备说明 |
| `scripts/` | 必交 | 快速验证、正式训练、比例扫描和成品训练脚本 |
| `实验报告.pdf` | 必交 | 老师最容易打开查看的报告版本 |
| `实验报告.md` | 建议提交 | 报告源文件，便于追溯图片和表格 |
| `提交说明.md` | 建议提交 | 本文件，说明提交包取舍 |
| `results/` 关键文件 | 必交 | 只保留报告引用的结果图和 CSV 即可 |
| `data/style_images/style.jpg` | 条件提交 | 若要求项目可直接运行，建议提交 |
| `data/test_images/test.jpg`, `test1.JPG`, `test2.png` | 条件提交 | 若要求项目可直接运行，建议提交 |
| `checkpoints/` | 条件提交 | 若老师要求权重，提交真实 `.pth` 或用 Git LFS/附件；否则可只说明可重训 |
| `logs/` 关键 CSV | 可选 | 若想增强可复核性，可提交 `ratio_*.csv` 和 `ratio_sweep.out` |

## 三、不建议提交的内容

| 内容 | 原因 |
|---|---|
| `data/downloads/` | Kaggle 原始下载数据，体积大，文件多，不是核心交付物 |
| 完整 `data/train_images/` | 训练集可通过脚本下载/复制，提交会显著增大作业包 |
| `archive/` | 过程文档和旧总结，不属于最终交付核心 |
| `.claude/` | AI/工具配置，与课程实验无关 |
| `sync_github.sh` | 仓库同步脚本，与实验本身无关 |
| `goal.png` | 若 README/报告未引用，可不提交 |
| `__pycache__/`, `.venv/`, `venv/` | 本地运行缓存或虚拟环境，不应提交 |

## 五、提交前检查清单

- [ ] `README.md` 中的数据数量、正式参数与报告一致。
- [ ] `实验报告.md` 中引用的图片都存在于 `results/`。
- [ ] `实验报告.pdf` 已由最新 Markdown 导出。
- [ ] 若提交精简包，已排除 `data/downloads/` 和完整训练集。
- [ ] 若要求可运行，已保留风格图和测试图，并说明如何准备训练集。
- [ ] 若要求提交模型权重，确认 `.pth` 是真实权重文件，不只是 Git LFS 指针。
- [ ] 压缩包内没有虚拟环境、缓存、AI 工具配置或旧归档文档。

## 六、可复现说明

精简提交包不包含完整训练数据和大权重文件时，复现实验按以下顺序执行：

```bash
pip install -r requirements.txt
bash scripts/download_training_data.sh
bash scripts/ratio_sweep.sh
python test.py --model checkpoints/experiments/ratio_250to1.pth \
  --input data/test_images/test.jpg \
  --output results/output.png \
  --style-image data/style_images/style.jpg \
  --image-size 672
```

其中 `scripts/ratio_sweep.sh` 会重新生成三组比例扫描模型、日志和关键对比结果。
